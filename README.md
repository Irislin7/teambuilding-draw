# 🎲 2026台州团建 · 今晚和谁吃？

> 144人随机抽取话题关键词的晚宴分桌系统

---

## 🚀 10分钟部署（照着做就行）

### 第一步：创建 Firebase 数据库（5分钟，免费）

1. 打开 https://console.firebase.google.com/ （用 Google 账号登录）
2. 点「添加项目」→ 名称填 `teambuilding-2026` → 一路下一步到创建完成
3. 左侧菜单「构建」→「Realtime Database」→「创建数据库」→ 区域选 `asia-southeast1` → **测试模式** → 启用
4. 点左上角齿轮 ⚙️ →「项目设置」→ 下滑到「您的应用」→ 点网页图标 `</>`
5. 应用名随便填 → 复制弹出的 `firebaseConfig` 那段代码（7行配置）

### 第二步：粘贴配置（1分钟）

用记事本打开 `index.html` 和 `admin.html`，找到这一行：
```
const firebaseConfig = { apiKey: "YOUR_API_KEY", ...
```
替换为你第一步复制的配置。两个文件都要替换。

### 第三步：设置安全规则（1分钟）

回到 Firebase 控制台 → Realtime Database → 上方「规则」标签 → 粘贴以下内容 → 点「发布」：

```json
{
  "rules": {
    "drawn": {
      ".read": true,
      "$index": {
        ".write": "!data.exists()"
      }
    }
  }
}
```

### 第四步：上线（3分钟）

**最简方式 — Netlify 拖拽部署：**
1. 打开 https://app.netlify.com/drop
2. 把 `抽签系统` 这个文件夹直接拖进去
3. 等10秒，得到一个链接（如 `https://xxx.netlify.app`）
4. 把链接分享到工作群 ✅

> 也可以用 GitHub Pages / Vercel / 公司内部服务器，只要能通过链接访问 HTML 就行。

## ❓ 常见问题

**Q: 会有两个人抽到同一个关键词吗？**
A: 不会。Firebase 保证原子性，每个关键词只能被领取一次。

**Q: 有人清除缓存能再抽一次吗？**
A: 清缓存后确实可以再抽，但只会抽到剩余未被领取的关键词（不影响其他人）。

**Q: 144人都抽完后会怎样？**
A: 按钮变灰，显示「名额已抽完」。

**Q: 活动当天网络不好怎么办？**
A: 只要能打开页面就行，抽签操作只需极少流量。

**Q: 如何生成二维码？**
A: 复制你的链接 → 打开 https://cli.im/ → 粘贴 → 下载二维码图片

## 📁 文件说明

A: 能，但进入"演示模式"：随机展示一个关键词，不保证唯一性。适合提前测试效果。

**Q: 费用？**
A: 完全免费。Firebase 免费额度：10GB/月流量、1GB存储、100个同时连接——对144人绰绰有余。
